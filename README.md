# 3.6 Практическая работа: Создание и запуск Django-приложения

## Цели практической работы
1. Создать и запустить Django-приложение.
2. По очереди описать модели.
3. Сгенерировать и выполнить миграции для созданных моделей (по отдельности).
4. Создать связи между таблицами.
5. Создать веб-страницы на основе Django-шаблонов.
6. Вывести на страницу сущности из базы данных.

---

## Что нужно сделать

### Проект `mysite`
1. Создать Django-проект с именем `mysite`.
2. Добавить приложение `shopapp` в проект.

---

### Приложение `shopapp`

#### Модель `Product`
Создать модель `Product` с полями типов:
- **CharField** — для названия продукта.
- **TextField** — для описания продукта.
- **DecimalField** — для цены.
- **PositiveSmallIntegerField** — для количества на складе.
- **DateTimeField** — для даты добавления.
- **BooleanField** — для признака активности.

#### Модель `Order`
Создать модель `Order` с полями типов:
- **TextField** — для описания заказа.
- **CharField** — для статуса заказа.
- **DateTimeField** — для даты создания.
- **ForeignKey** — связь с моделью `User`.
- **ManyToManyField** — связь с моделью `Product`.

---

### Миграции
1. Выполнить стандартные миграции.
2. Создать отдельные миграции для каждой из моделей:
    - `Product`
    - `Order`

---

### Пользователь
- Создать администратора через команду `createsuperuser`.

---

### Management Commands
1. **Создание продуктов**:
    - Создать команду в `shopapp` для добавления продуктов через `get_or_create`.
    - Имя команды: `create_products.py`.

2. **Создание заказов**:
    - Создать команду в `shopapp` для добавления заказов через `get_or_create`.
    - Имя команды: `create_orders.py`.
    - Установить связи между заказами и существующими продуктами.

---

### View-функции
1. **Shop index**: страница со списком ссылок на доступные страницы.
2. **Products list**: страница со списком продуктов.
3. **Orders list**: страница со списком заказов с загруженными пользователями и продуктами.

#### Подключение View-функций
- Настроить маршруты в `urls.py` внутри приложения `shopapp`.

---

## Ожидаемый результат
После выполнения практической работы вы получите работающий Django-проект, включающий:
- Приложение `shopapp` с моделями `Product` и `Order`.
- Страницы для отображения данных из базы:
    - Список продуктов.
    - Список заказов.
- Management команды для генерации данных.


## План решения задачи

1. **Подготовка окружения**
    - Установить Python и создать виртуальное окружение.
    - Установить Django.

2. **Создание проекта**
    - Создать Django-проект с именем `mysite`.
    - Запустить сервер для проверки.

3. **Создание приложения**
    - Создать приложение `shopapp`.
    - Зарегистрировать приложение в настройках проекта.

4. **Разработка моделей**
    - Описать модели `Product` и `Order` в файле `models.py`.
    - Сгенерировать и выполнить миграции для моделей.

5. **Создание администратора**
    - Создать суперпользователя через команду `createsuperuser`.
    - Убедиться в доступе к админке.

6. **Создание Management команд**
    - Реализовать команду для создания продуктов (`create_products.py`).
    - Реализовать команду для создания заказов (`create_orders.py`) и связи заказов с продуктами.

7. **Реализация View-функций**
    - Разработать View-функции для отображения списка ссылок, продуктов и заказов.
    - Настроить маршруты в `urls.py`.

8. **Создание шаблонов**
    - Разработать HTML-шаблоны для страниц:
        - Список ссылок.
        - Список продуктов.
        - Список заказов.

9. **Тестирование**
    - Проверить работу всех страниц.
    - Убедиться, что данные корректно отображаются на веб-страницах.

10. **Финальная проверка**
    - Провести общее тестирование приложения.
    - Подготовить проект к сдаче.
